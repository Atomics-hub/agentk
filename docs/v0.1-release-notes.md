# AgentK v0.1.0 Release Notes

This records the published GitHub release body and final release evidence for
`v0.1.0`.

## Summary

AgentK v0.1.0 is a firewall and flight recorder for AI agents. It sits between
an MCP client and a local subprocess stdio MCP server, mediates covered
boundary crossings through policy, records provenance, and writes replayable
evidence.

The release claim is intentionally narrow: AgentK lets a powerful agent keep
broad tool access while high-risk transitions still require policy,
provenance, and evidence.

## Implemented Highlights

- One-command MCP killer demo where poisoned MCP output attempts secret
  exfiltration and an unsafe patch, and AgentK blocks both follow-up actions.
- Before/after MCP shim eval scorecard comparing baseline passthrough with
  AgentK mediation.
- Local subprocess stdio MCP proxy for tools, resources, prompts, lifecycle
  gates, bounded IO, downstream timeout/close handling, error redaction,
  default-deny unsupported methods, and optional trace output.
- Hash-first evidence for MCP descriptors, requests, responses, client intent,
  and denied transitions without recording raw poisoned payloads.
- `trace-inspect`, `replay`, `fork-replay`, behavior fork replay, and signature
  verification output for reviewer-visible evidence.
- Release-audit coverage for poisoned-output blocking, public MCP interop
  transcripts, timeout and close transcripts, notification no-passthrough,
  malformed downstream shapes, resource subscription no-passthrough, signature
  signer summaries, and secret-reference redaction.

## Reviewer Commands

```sh
cargo run --locked -- mcp-killer-demo
cargo run --locked -- mcp-shim-eval
cargo run --locked -- trace-inspect .agentk/runs/mcp-killer-demo.jsonl
cargo run --locked -- replay .agentk/runs/mcp-killer-demo.jsonl
cargo run --locked -- release-audit
```

For the final release gate, the maintainer should run strict release-audit with
a durable signing key outside the repository:

```sh
AGENTK_REQUIRE_SIGNING_KEY=1 \
AGENTK_RELEASE_REMOTE_APPROVED=1 \
AGENTK_SIGNING_KEY_FILE=<path-outside-repo> \
cargo run --locked -- release-audit --strict
```

## Accepted v0.1 Limits

These limits are explicit release scope, not hidden caveats:

- The operator-ready path is the local subprocess stdio MCP proxy.
- Hosted, remote, or packaged production MCP transport is deferred.
- Production signing-key custody is deferred. v0.1 supports configured local
  signing keys for release gates but does not claim HSM, KMS, or secret-manager
  custody.
- Production secret storage is deferred. v0.1 supports external secret
  references and metadata-only availability checks, not secret retrieval from
  production stores.
- OS sandboxing, eBPF, cgroups, and filesystem isolation are deferred.
- MCP resource subscription flows are unsupported until an explicit policy
  contract exists; unsupported subscription requests fail closed rather than
  becoming generic passthrough.

## Not Claimed

AgentK v0.1.0 is not:

- production-ready secret storage
- production signing-key custody
- a hosted MCP gateway
- a complete MCP protocol implementation
- an OS sandbox, eBPF monitor, cgroup controller, or filesystem isolation layer
- an agent framework, planner, memory system, or orchestrator
