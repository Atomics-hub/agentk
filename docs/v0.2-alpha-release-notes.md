# AgentK v0.2 Alpha Release Notes Draft

This is the release-notes draft for the next AgentK team-sidecar alpha. It is
not final release evidence until the release commit, strict release-audit
result, package archive checksum, package release manifest, signed tag
verification, and signer evidence are filled in.

## Summary

AgentK v0.2 alpha turns the v0.1 public prototype into an installable local
team sidecar release candidate.

The release claim is still intentionally narrow: AgentK is an MCP action
firewall and flight recorder. It gives a team a deployable sidecar in front of
Claude, Codex, Cursor, or another MCP client; records redacted evidence for
tool/resource/prompt boundaries; exposes a local approvals and audit dashboard;
and packages a no-credential safe-agent demo across GitHub/Postgres/Slack/
filesystem-shaped workflows.

## Implemented Highlights

- Installable team sidecar package with `manifest.json`, `package.lock.json`,
  stable launcher scripts, Claude Desktop and generic Codex/Cursor snippets,
  systemd/launchd service templates, Dockerfile/Compose templates, Caddy/nginx
  reverse-proxy templates, dummy env examples, package self-checks, and saved
  package-check plus HTTP/team handoff JSON release evidence.
- Package archive and receiver-side handoff flow:
  `sidecar-package --archive-out`, `sidecar-package-archive-check`,
  `sidecar-package-install`, `sidecar-package-release-manifest`, and
  `sidecar-package-release-manifest-check`.
- Packaged release-candidate smoke gate that recreates the package/archive,
  verifies the archive checksum, installs the package, writes the release
  manifest, runs the packaged safe-agent demo, renders the dashboard, refreshes
  durable stores, writes the compact operator handoff JSON/Markdown artifact,
  runs the sidecar doctor support/remediation report, writes deploy handoff
  JSON/Markdown evidence, writes permissions handoff JSON/Markdown evidence,
  writes production preflight JSON/Markdown evidence, exports notification
  payloads, dry-runs bridge delivery, preflights Postgres push, and can write
  a release-ticket JSON evidence report with
  SHA-256/byte counts for required handoff files.
- Release evidence checker:
  `release-evidence-check` re-validates a saved release-candidate smoke JSON
  report against the current package, archive, release manifest, dashboard,
  store, notification, and handoff artifact byte counts and SHA-256s.
- Final release handoff report:
  `release-finalize` writes one offline JSON report binding the release commit,
  release notes hash, package archive SHA-256, package release manifest, saved
  smoke evidence, active evidence-signing public key, worktree state, and
  optional signed tag verification. It does not tag, push, upload assets, or
  publish a GitHub release.
- Offline release ticket bundle:
  `release-ticket` composes release status, packaged RC smoke evidence,
  evidence-check results, explicit product-objective coverage checks, and
  accepted alpha limit checks plus release finalization into one local reviewer
  handoff directory without tagging, pushing, uploading assets, or publishing.
  The ticket checks install/package provenance for archive checksum,
  release-manifest binding, install receipt, package lock, launchers, client
  snippets, deploy templates, and package self-check evidence.
  The ticket checks store/notification handoff evidence for durable approvals,
  Postgres load coverage, Slack/GitHub/email redacted payloads, and local
  env-held bridge config.
  The ticket checks served dashboard runtime evidence for launcher package
  preflight, loopback/admin-token defaults, bounded request caps, supervisor
  hardening, redacted probes, and permission-checked review APIs.
  The ticket checks quickstart handoff evidence for first-run package health,
  HTTP/team handoff, demo, deploy, support, permissions, preflight, client,
  dashboard, owner-specific next actions, artifact inventory, and local
  non-hosted scope.
  The ticket also checks support/doctor handoff evidence for operator handoff
  refresh, no-remediation doctor output, release-manifest binding, hashed
  support inventory, and local non-hosted scope.
  The ticket checks Homebrew handoff evidence for local formula generation,
  source archive SHA verification, tap checkout byte match, dirty-path hygiene,
  no tap publication, and top-level artifact inventory entries spanning the
  release status, smoke evidence, finalization, package, quickstart, dashboard,
  client, permissions, deploy, support, demo, store, notification, service
  template, and Homebrew handoff files.
  The MCP gateway objective checks the saved HTTP/SSE handoff proof for
  loopback defaults, auth-token env handoff, bounded replay, Last-Event-ID
  resume, and no hosted control plane. The dashboard objective checks the saved
  dashboard handoff proof for static dashboard readiness, reviewer-scoped team
  store evidence, open approvals, notifications, dashboard env handoff, and
  local non-hosted scope. The multi-user permissions objective checks reviewer
  roles, identity mapping coverage, reviewer-scoped reads, authorized approval
  recording, fail-closed unauthorized reviewer rejection, and local non-hosted
  scope. The Claude/Codex/Cursor objective checks packaged client snippets,
  stdio/TCP/HTTP launchers, Streamable HTTP handoff evidence, and local
  non-hosted scope. The deploy/preflight handoff check validates deploy
  templates, supervisor env examples, secret-reference placeholders, non-local
  bind defaults, no live secret retrieval, and local non-hosted scope.
- Release publication preflight:
  `release-publication-check` verifies the strict finalization report,
  production-ready evidence signer, signed-tag evidence, package archive hash,
  package release-manifest path, and final release-note evidence fields before a
  maintainer creates the GitHub release page. It does not tag, push, upload
  assets, or publish a release.
- Local approvals/audit dashboard with permission-checked approve/deny APIs,
  role-aware reviewer views, requester-scoped views from signed agent identity,
  redacted readiness/metrics probes, admin token support, reviewer token
  support, strict query/header validation, and durable store refresh.
- Multi-user local permission model with scoped reviewers, roles, reviewer
  tokens, append-only decisions, and external IdP group-to-reviewer mapping
  validation that omits issuer, audience, and claim values from reports.
- Durable team store with redacted current JSON views plus normalized JSONL and
  Postgres-loadable summary rows for traces, audit events, approvals, blocked
  rules, syscall summaries, evidence references, reviewers, identity mappings,
  and notification outbox rows.
- Local notification bridges for approval events:
  Slack payload export/dry-run send, GitHub issue payload export/dry-run send,
  and sendmail-ready email payload export/dry-run send. Delivery credentials
  remain environment-held and are not stored in AgentK payload artifacts.
- Bounded local MCP gateway transports beyond stdio: TCP JSON-RPC and
  Streamable HTTP POST adapters with session limits, idle reaping, request
  size bounds, origin handling, optional bearer auth, readiness probes,
  sanitized errors, and no-passthrough hardening for unsupported or malformed
  traffic.
- Safe-agent demo packaging that runs without live credentials and proves the
  GitHub/Postgres/Slack/filesystem workflow through blocked risky actions,
  allowed safe actions, redacted trace-inspect evidence, dashboard review, and
  store export.
- Packaged operator handoff mode:
  `agentk-sidecar-ops-handoff` refreshes the packaged safe-agent demo,
  dashboard, exported store, durable team store, notification payload drafts,
  team permissions, and identity summary, then writes one archiveable
  `operator-handoff.json` and Markdown summary for local/team release review.
- Packaged sidecar doctor:
  `agentk-sidecar-doctor` diagnoses an unpacked, copied, or updated package;
  verifies launchers, dummy env templates, bounded HTTP/SSE handoff readiness,
  dashboard/store readiness, install receipt provenance, operator handoff
  artifacts, evidence retention, optional release-manifest binding, and
  safe-agent demo integrity; then writes support-ready `sidecar-doctor.json`
  and Markdown remediation output.
- Packaged support bundle:
  `agentk-sidecar-support-bundle` refreshes the operator handoff, runs the
  sidecar doctor, and writes one archiveable support-bundle JSON/Markdown report
  with hashed package, dashboard, store, trace, and notification evidence.
- Packaged safe-agent demo handoff:
  `agentk-sidecar-demo-handoff` refreshes the credential-free
  GitHub/Postgres/Slack/filesystem demo evidence and writes one archiveable
  demo-handoff JSON/Markdown report with hashed trace, dashboard, durable
  store, and notification payload artifacts for onboarding review.
- Packaged sidecar quickstart:
  `agentk-sidecar-quickstart` runs package health, HTTP/team handoff, demo
  handoff, deploy handoff, support bundle, permissions handoff, production
  preflight, client handoff, and dashboard handoff checks, then writes one
  first-run quickstart JSON/Markdown report for team onboarding.
- Release hardening from recent merged PRs including HTTP method override
  rejection, proxy/trace method rejection, duplicate dashboard `Content-Type`
  rejection, WebSocket handshake rejection, content encoding rejection,
  private-network preflight rejection, sanitized MCP HTTP downstream failure
  responses and metrics, safe-agent evidence surfacing, package binary
  preflight, and normalized store evidence summaries.

Run the Homebrew formula and tap handoff checks only when a tap handoff is part
of the release. They verify the local Ruby formula, source archive hash, exact
tap checkout formula bytes, and unrelated dirty-file hygiene; they do not
publish a tap.

For the final release gate, the maintainer should run strict release-audit with
a durable signing key outside the repository:

```sh
AGENTK_REQUIRE_SIGNING_KEY=1 \
AGENTK_RELEASE_REMOTE_APPROVED=1 \
AGENTK_SIGNING_KEY_FILE=<path-outside-repo> \
cargo run --locked -- release-audit --strict
```

The reviewer handoff should also include the generated package release
manifest, archive checksum, and bounded local HTTP/SSE handoff contract:

```sh
cargo run --locked -- sidecar-package-release-manifest \
  --package installed/agentk-sidecar \
  --archive dist/agentk-sidecar.tar \
  --out dist/agentk-sidecar-release-manifest.json \
  --force \
  --json
cargo run --locked -- sidecar-package-http-handoff-check \
  --root installed/agentk-sidecar \
  --json
```

## Accepted v0.2 Alpha Limits

These limits are explicit alpha scope, not hidden caveats:

- The production-ready path is still local sidecar deployment. Hosted control
  plane, managed multi-tenant service, and hosted release operations are
  deferred.
- TCP and Streamable HTTP are bounded local adapters. AgentK does not yet claim
  public internet gateway deployment, TLS termination, or hosted SSE streaming.
  Reverse-proxy metadata remains an explicit reviewed opt-in: trusted-proxy mode
  accepts only clean forwarded headers and reports redacted counts, not a full
  hosted edge or TLS boundary. The current Streamable HTTP/SSE reviewer handoff
  is the finite authenticated buffered local alpha documented in
  `clients/http-sse-handoff.md`.
- The packaged approval/audit handoff is a local/team sidecar alpha documented
  in `clients/team-audit-dashboard-handoff.md` and validated by
  `agentk-sidecar-team-handoff-check`. The packaged
  `agentk-sidecar-ops-handoff` command adds one JSON/Markdown operator handoff
  artifact for the same local proof. The packaged `agentk-sidecar-doctor`
  command adds one support/remediation report for install or update checks and
  can bind that report to the package release manifest. The packaged
  `agentk-sidecar-support-bundle` command combines the handoff, doctor, and
  hashed evidence inventory into one archiveable support artifact. The packaged
  `agentk-sidecar-dashboard-handoff` command refreshes the demo trace, renders
  the static dashboard, syncs the durable team store, and writes one
  JSON/Markdown dashboard readiness artifact. The
  `sidecar-package-release-manifest-check` command re-verifies that manifest
  against relocated package, archive, checksum, and install receipt files.
  These prove GitHub/Postgres/Slack/filesystem safe-agent demo evidence,
  including blocked filesystem patch proof, reviewer-scoped dashboard review,
  durable team-store sync/check, notification outbox handoff, and package
  support readiness, not hosted SaaS.
- `release-publication-check` is a local preflight for a maintainer-operated
  GitHub release page. It verifies final evidence consistency, but it does not
  create tags, upload assets, publish a release, or assert that a hosted release
  operation has already happened.
- External identity is mapping and validation only. AgentK does not yet verify
  live OIDC/JWT tokens or contact an IdP during approval decisions.
- Production secret-provider references are validated for handoff shape, but
  AgentK still does not retrieve production secret bytes from AWS, GCP, Azure,
  Vault, or 1Password.
- Notification bridges are local bridge processes using env-held delivery
  credentials. Hosted ticketing integrations and managed notification delivery
  are deferred.
- OS sandboxing, eBPF, cgroups, and filesystem isolation remain deferred.
- The dashboard is a local/team review surface, not a hosted control plane.

## Not Claimed

AgentK v0.2 alpha is not:

- a hosted SaaS control plane
- a public internet MCP gateway
- production secret storage or KMS/HSM custody
- live IdP authentication middleware
- a ticketing system
- an OS sandbox, eBPF monitor, cgroup controller, or filesystem isolation layer
- an agent framework, planner, memory system, or orchestrator
