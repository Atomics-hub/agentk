# AgentK v0.2 Alpha Release Notes Draft

This is the release-notes draft for the next AgentK team-sidecar alpha. It is
not final release evidence until the release commit, strict release-audit
result, package archive checksum, package release manifest, signed tag
verification, and signer evidence are filled in.

## Summary

AgentK v0.2 alpha turns the v0.1 public prototype into an installable local
team sidecar release candidate.

The release claim is still intentionally narrow: AgentK is an MCP action
firewall and flight recorder. It gives a team a deployable sidecar in front of
Claude, Codex, Cursor, or another MCP client; records redacted evidence for
tool/resource/prompt boundaries; exposes a local approvals and audit dashboard;
and packages a no-credential safe-agent demo across GitHub/Postgres/Slack/
filesystem-shaped workflows.

## Implemented Highlights

- Installable team sidecar package with `manifest.json`, `package.lock.json`,
  stable launcher scripts, Claude Desktop and generic Codex/Cursor snippets,
  systemd/launchd/Docker Compose templates, dummy env examples, and package
  self-checks.
- Package archive and receiver-side handoff flow:
  `sidecar-package --archive-out`, `sidecar-package-archive-check`,
  `sidecar-package-install`, and `sidecar-package-release-manifest`.
- Packaged release-candidate smoke gate that recreates the package/archive,
  verifies the archive checksum, installs the package, writes the release
  manifest, runs the packaged safe-agent demo, renders the dashboard, refreshes
  durable stores, exports notification payloads, dry-runs bridge delivery, and
  preflights Postgres push.
- Local approvals/audit dashboard with permission-checked approve/deny APIs,
  role-aware reviewer views, requester-scoped views from signed agent identity,
  redacted readiness/metrics probes, admin token support, reviewer token
  support, strict query/header validation, and durable store refresh.
- Multi-user local permission model with scoped reviewers, roles, reviewer
  tokens, append-only decisions, and external IdP group-to-reviewer mapping
  validation that omits issuer, audience, and claim values from reports.
- Durable team store with redacted current JSON views plus normalized JSONL and
  Postgres-loadable summary rows for traces, audit events, approvals, blocked
  rules, syscall summaries, evidence references, reviewers, identity mappings,
  and notification outbox rows.
- Local notification bridges for approval events:
  Slack payload export/dry-run send, GitHub issue payload export/dry-run send,
  and sendmail-ready email payload export/dry-run send. Delivery credentials
  remain environment-held and are not stored in AgentK payload artifacts.
- Bounded local MCP gateway transports beyond stdio: TCP JSON-RPC and
  Streamable HTTP POST adapters with session limits, idle reaping, request
  size bounds, origin handling, optional bearer auth, readiness probes,
  sanitized errors, and no-passthrough hardening for unsupported or malformed
  traffic.
- Safe-agent demo packaging that runs without live credentials and proves the
  GitHub/Postgres/Slack/filesystem workflow through blocked risky actions,
  allowed safe actions, redacted trace-inspect evidence, dashboard review, and
  store export.
- Release hardening from recent merged PRs including HTTP method override
  rejection, proxy/trace method rejection, duplicate dashboard `Content-Type`
  rejection, WebSocket handshake rejection, content encoding rejection,
  private-network preflight rejection, safe-agent evidence surfacing, package
  binary preflight, and normalized store evidence summaries.

## Reviewer Commands

```sh
cargo run --locked -- release-status --json
cargo run --locked -- release-audit
cargo run --locked -- release-candidate-smoke --json
cargo run --locked -- sidecar-package-http-handoff-check --root dist/agentk-sidecar --json
cargo test --locked
git diff --check
git status --short
```

For the final release gate, the maintainer should run strict release-audit with
a durable signing key outside the repository:

```sh
AGENTK_REQUIRE_SIGNING_KEY=1 \
AGENTK_RELEASE_REMOTE_APPROVED=1 \
AGENTK_SIGNING_KEY_FILE=<path-outside-repo> \
cargo run --locked -- release-audit --strict
```

The reviewer handoff should also include the generated package release
manifest, archive checksum, and bounded local HTTP/SSE handoff contract:

```sh
cargo run --locked -- sidecar-package-release-manifest \
  --package installed/agentk-sidecar \
  --archive dist/agentk-sidecar.tar \
  --out dist/agentk-sidecar-release-manifest.json \
  --force \
  --json
cargo run --locked -- sidecar-package-http-handoff-check \
  --root installed/agentk-sidecar \
  --json
```

## Accepted v0.2 Alpha Limits

These limits are explicit alpha scope, not hidden caveats:

- The production-ready path is still local sidecar deployment. Hosted control
  plane, managed multi-tenant service, and hosted release operations are
  deferred.
- TCP and Streamable HTTP are bounded local adapters. AgentK does not yet claim
  public internet gateway deployment, TLS termination, reverse-proxy trust, or
  hosted SSE streaming. The current Streamable HTTP/SSE reviewer handoff is the
  finite authenticated buffered local alpha documented in
  `clients/http-sse-handoff.md`.
- The packaged approval/audit handoff is a local/team sidecar alpha documented
  in `clients/team-audit-dashboard-handoff.md` and validated by
  `agentk-sidecar-team-handoff-check`. It proves safe-agent demo evidence,
  reviewer-scoped dashboard review, durable team-store sync/check, and
  notification outbox handoff, not hosted SaaS.
- External identity is mapping and validation only. AgentK does not yet verify
  live OIDC/JWT tokens or contact an IdP during approval decisions.
- Production secret-provider references are validated for handoff shape, but
  AgentK still does not retrieve production secret bytes from AWS, GCP, Azure,
  Vault, or 1Password.
- Notification bridges are local bridge processes using env-held delivery
  credentials. Hosted ticketing integrations and managed notification delivery
  are deferred.
- OS sandboxing, eBPF, cgroups, and filesystem isolation remain deferred.
- The dashboard is a local/team review surface, not a hosted control plane.

## Not Claimed

AgentK v0.2 alpha is not:

- a hosted SaaS control plane
- a public internet MCP gateway
- production secret storage or KMS/HSM custody
- live IdP authentication middleware
- a ticketing system
- an OS sandbox, eBPF monitor, cgroup controller, or filesystem isolation layer
- an agent framework, planner, memory system, or orchestrator
